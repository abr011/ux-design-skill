UX Design System

Opinionated design system for building kind, smart web interfaces. Vanilla CSS, no frameworks, no dependencies.

Design Philosophy

Three principles guide every design decision:

Kind

  • Error messages with empathy: "Please check the @ and domain" not "Error!"
  • User-first thinking: Respect the user's time and attention
  • Friendly, not childish: Professional without corporate bullshit

Smart

  • Data-driven decisions: Measure before redesigning, validate after
  • Modern UX patterns: Lazy registration, forgiving software, progressive disclosure
  • Know when to break rules: Patterns serve users, not the other way around

Experienced

  • Reduction over addition: The best UX is the one users don't remember
  • Respect user context: Don't interrupt flow, don't demand attention
  • Progressive disclosure: Show what's needed now, reveal the rest when relevant

Core Question

"What to remove?" not "What to add?"

Every field is an opportunity to abandon. Every step is a friction point. Every duplication is an annoyance.

1. Forgiving Software

No confirmation dialogs. Instead, use reversible actions with undo:

Live Example
Item archived. 5s
  • Actions are reversible for ~5 seconds via toast
  • "Deleted" items go to Archive (not permanently removed)
  • Archive page shows all archived items with restore option

2. Data Display

Cards for browsing (visual scanning). Lists for detailed/searchable views. No tables for data display.

  • Cards: flex-direction: column, flex-grow: 1 on body pushes footer to bottom
  • No borders between cards — use whitespace (gap) only

3. Form Inputs

Live Example
Email
Password
  • Clear bordered inputs with visible focus states
  • Labels above inputs, never inline
  • Input width reflects expected content length

4. Dropdown Avoidance

DO < 5 options: use radio buttons or selectable pills

OK 5+ options: dropdown acceptable, but make items rich

Pills instead of dropdown (3 options)

5. Buttons

Button variants
  • Primary: Indigo/purple (--color-primary)
  • Secondary: Gray outlined
  • Danger: Red (only truly destructive, rare)
  • NEVER Orange buttons

6. Minimalist Design

  • Clean, simple interfaces with generous whitespace
  • No pure colors: off-white (#fafafa) and soft black (#1f2937)
  • No borders/lines for separation: whitespace only
  • No icons/emoji unless they have explicit functional meaning

7. Heading or Separator, Never Both

Headings and separators both create visual boundaries. Use one or the other.

CORRECT

Section Header

Content below heading with whitespace only

WRONG

Section Header

Heading + border = redundant double-signaling

8. Component Consistency

When modifying a reusable visual pattern, audit ALL occurrences across the app.

  • Search codebase for all places a component appears
  • Apply same font sizes, spacing, colors to every occurrence
  • Document intentional differences

9. Visual Audit Page

Every UI project should have audit-cards.html — standalone HTML rendering every component variant with the app's real CSS.

10. No System Popups

BANNED alert(), confirm(), prompt()

Use inline UI instead: toast notifications, modal sheets, inline forms.
